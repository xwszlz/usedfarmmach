# 产品封面图优化方案

## 现状分析

基于对当前数据库（开发环境）的分析，我们有以下发现：

### 数据概览
- **产品总数**: 27个
- **有图片的产品**: 27个（100%）
- **平均图片数量**: 每个产品10张图片
- **当前封面图选择逻辑**: 使用 `sortOrder` 最小的图片作为封面图（通常是 `1.jpg`）

### 当前问题识别
1. **封面图选择简单化**: 目前仅按文件名数字顺序（1.jpg, 2.jpg...）排序，未考虑图片内容质量
2. **缺乏选图标准**: 没有明确的封面图选择标准，可能导致以下问题：
   - 局部细节图被用作封面，无法展示整机外观
   - 缺少品牌标识展示
   - 图片质量（光线、角度、背景）参差不齐
   - 无法凸显产品卖点

3. **用户反馈**: 用户明确表示"封面图不够漂亮"，需要：
   - 最大程度反映整机状况的图片
   - 带有品牌标识的图片

## 选图标准（推荐）

### 一级标准（必须满足）
1. **整机展示**: 完整展示农机设备的全貌，避免局部特写
2. **品牌标识可见**: 图片中应清晰显示品牌Logo或标识
3. **光线充足**: 图片明亮，无过度阴影或曝光
4. **角度适宜**: 正面或斜45度角，展示设备主要特征
5. **背景整洁**: 背景简洁，无杂乱物品干扰视线

### 二级标准（优化加分项）
1. **多设备场景**: 展示多台设备排列，体现规模（如用户提供的甜菜机全景图）
2. **工作状态**: 设备在工作中的场景图
3. **新机感受**: 设备清洁、外观新颖，给人"如新"感觉
4. **品牌元素突出**: 品牌标识位于图片显著位置
5. **高清画质**: 图片分辨率高，细节清晰

### 规避类型
1. **局部细节图**: 仅展示引擎、轮胎等局部
2. **低质量图片**: 模糊、光线暗、像素低
3. **无关背景**: 背景杂乱，分散注意力
4. **无品牌标识**: 无法识别品牌
5. **角度怪异**: 难以理解设备整体结构

## 优化方案

### 方案一：人工筛选 + 数据库更新（快速见效）
1. **建立评审小组**: 由运营/产品人员组成，负责图片筛选
2. **制定评审表格**: 基于上述标准制作评分表
3. **分批处理**: 优先处理重点产品（热门、高价值）
4. **数据库更新**: 更新 `ProductImage` 表中的 `sortOrder`，将最佳图片设为 `sortOrder=0`
5. **时间预估**: 2-3天完成27个产品的筛选

### 方案二：开发封面图管理工具（中长期）
1. **开发管理界面**: 在Admin后台添加封面图管理功能
2. **功能特性**:
   - 按产品浏览所有图片
   - 点击选择封面图
   - 按标准评分系统自动推荐
   - 批量操作支持
3. **技术实现**: 
   - 前端: React组件，图片网格展示
   - 后端: API接口更新 `sortOrder` 或 `isPrimary`
   - 数据库: 增加 `isCover` 标志字段（可选）
4. **开发周期**: 1-2周

### 方案三：AI辅助选图（未来扩展）
1. **图像分析模型**: 使用计算机视觉技术评估图片质量
2. **自动评分项目**:
   - 整机完整性检测
   - 品牌Logo识别
   - 图片清晰度评分
   - 光线与背景评估
3. **集成路径**: 可作为管理工具的智能推荐模块

## 技术实现细节

### 数据库结构调整（可选）
```prisma
// 在 ProductImage 模型中增加 coverScore 字段
model ProductImage {
  id        String  @id @default(cuid())
  productId String
  url       String
  sortOrder Int     @default(0)
  isPrimary Boolean @default(false)
  coverScore Float?   @default(0.0) // 封面图评分，0-100
  coverReason String? // 选为封面图的理由
  
  product Product @relation(fields: [productId], references: [id], onDelete: Cascade)
  @@index([productId])
}
```

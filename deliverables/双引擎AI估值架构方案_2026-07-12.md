# 双引擎AI估值架构方案

> 日期：2026-07-12
> 状态：方案设计阶段
> 依赖：AI深度分析调研报告 + 中国农机补贴数据库分析报告

---

## 一、背景与动机

### 1.1 现状问题

当前估值系统（V4）是一套"万能公式"，用同一套逻辑处理所有品牌：

```
估值 = 基准价 × 品牌系数 × 年份因子 × 工时因子 × 成色因子 × 市场因子 × 规格因子 × 视觉因子 × 视频因子
```

**问题清单**：

| 问题 | 影响 |
|------|------|
| 基准价仅覆盖50个型号 | 国产机型大面积"默认80万"，估值严重失真 |
| 国产品牌系数一刀切 0.30-0.65 | 无法区分雷沃/沃得/东方红/牧神等不同档次 |
| 无补贴因素 | 国内二手价深受补贴影响（平均24.6%），完全忽略 |
| 无地区差价 | 同型号在山东vs贵州差价可达30%，未体现 |
| 无价格趋势 | 4年补贴数据有价格走势，完全未利用 |
| AI图片分析对国产机不精准 | 训练数据偏国际品牌，国产机成色判断有偏差 |

### 1.2 两套业务的本质差异

| 维度 | 国际品牌（出口） | 国内品牌（内贸+出口发展中国家） |
|------|-----------------|-------------------------------|
| **品牌** | 迪尔/克拉斯/凯斯/纽荷兰/麦赛等 | 东方红/雷沃/沃得/常发/牧神等 |
| **数据源** | 手动维护50型号基准价 | 865万条补贴真实交易 |
| **货币** | USD为主 | CNY为主 |
| **核心因子** | 品牌溢价+AI视觉成色+规格+出口物流 | 补贴残值+地区差价+品牌档次+价格趋势 |
| **AI作用** | 图片/视频成色分析（核心） | 基准价校验+异常检测（辅助） |
| **置信度基础** | AI视觉+规格完整度 | 数据匹配度+样本量 |
| **用户场景** | 跨境买家判断进口二手价 | 国内买家/卖家判断国产二手价 |

**结论：两套业务的数据基础、核心因子、AI角色完全不同，必须分引擎处理。**

## 二、架构设计

### 2.1 总体架构

```
                    POST /api/valuation
                          |
                  ValuationRouter
                 (品牌来源判定)
                    /        \
              国际品牌        国内品牌
                |              |
        InternationalEngine  DomesticEngine
           (V4 出口)          (V5 补贴)
                |              |
         ValuationResult    ValuationResult
         (engine: "V4")   (engine: "V5")
                \              /
                 \            /
              统一返回格式
```

### 2.2 路由判定逻辑

```typescript
// src/lib/valuation/router.ts

function determineEngine(brand: Brand, product: Product): "V4" | "V5" {
  // 1. 优先使用 Brand.isChineseBrand 字段
  if (brand.isChineseBrand === true) return "V5";
  if (brand.isChineseBrand === false) return "V4";

  // 2. 回退：品牌系数判定（0.70 为分水岭）
  const factor = getBrandFactor(brand.nameZh);
  if (factor <= 0.70) return "V5";  // 国产
  if (factor >= 0.80) return "V4";  // 进口

  // 3. 最终回退：品牌名关键词
  const chineseKeywords = ["东方红", "雷沃", "沃得", "常发", "牧神", "迈科", "德翔", "盈嘉"];
  if (chineseKeywords.some(kw => brand.nameZh.includes(kw))) return "V5";
  return "V4";
}
```

### 2.3 国际品牌出口估值引擎（V4 — 保持现有）

**不改动**，继续使用现有 V4 引擎：

```
估值 = 基准价 × 品牌系数(0.80-1.30) × 年份因子 × 工时因子 × 成色因子 × 市场因子 × 规格因子 × 视觉因子 × 视频因子
```

**优化方向**（后续迭代）：
- 修复 deep-analysis-card 前端改走内部 API
- 视频分析增加 Gemini 2.5 Flash 免费降级
- 补充更多国际型号基准价

### 2.4 国内品牌补贴估值引擎（V5 — 新建）

**全新公式**：

```
估值 = 补贴基准价 × 品牌档次系数 × 年份折旧 × 工时因子 × 成色因子 × 地区因子 × 补贴残值因子 × 趋势因子
```

#### 因子详解

| 因子 | 数据来源 | 计算方式 |
|------|----------|----------|
| **补贴基准价** | 865万条补贴数据聚合 | 按型号+品类取中位销售价（非补贴价）|
| **品牌档次系数** | 补贴数据按厂家分组 | 该厂家均价 / 品类均价，范围0.50-1.20 |
| **年份折旧** | 现有折旧表 + 补贴数据验证 | 0-3年5%/年, 4-7年6%/年, 8-12年5%/年, 13+年4% |
| **工时因子** | 现有逻辑 | 不变（预期500h/年，超扣低加）|
| **成色因子** | AI图片分析（辅助）+ 用户自选 | 与V4一致，但国产机AI权重降低 |
| **地区因子** | 25省份补贴数据 | 该省份均价 / 全国均价，范围0.85-1.15 |
| **补贴残值因子** | 补贴额/销售价比例 | 二手价 = 新机价 × (1 - 补贴比例 × 残值系数) |
| **趋势因子** | 2022-2025年度均价 | 当年均价 / 上年均价，范围0.92-1.08 |

#### V5 置信度计算

```typescript
function calcConfidenceV5(input, matchResult) {
  let confidence = 0.50; // 基准

  // 数据匹配精度
  if (matchResult.matchLevel === "exact_model") confidence += 0.25;    // 精确型号
  else if (matchResult.matchLevel === "model_substring") confidence += 0.18;
  else if (matchResult.matchLevel === "category_only") confidence += 0.10;

  // 样本量
  if (matchResult.sampleSize >= 100) confidence += 0.15;
  else if (matchResult.sampleSize >= 30) confidence += 0.10;
  else if (matchResult.sampleSize >= 10) confidence += 0.05;

  // AI视觉成色（辅助验证）
  if (input.visualResult) confidence += 0.08;

  // 规格完整度
  if (input.enginePower) confidence += 0.02;

  return Math.min(confidence, 0.95);
}
```

#### Phase 2：存储方案

**方案A：嵌入式JSON（推荐MVP用）**
- 聚合后约 5-10MB JSON 文件
- 部署时嵌入 `src/lib/valuation/subsidy-data.json`
- 内存查找，零数据库开销
- 适合 Vercel Serverless

**方案B：PostgreSQL表（后续扩展用）**
```sql
CREATE TABLE subsidy_model_price (
  id TEXT PRIMARY KEY,
  model TEXT,
  category TEXT,
  manufacturer TEXT,
  median_price REAL,
  avg_price REAL,
  min_price REAL,
  max_price REAL,
  subsidy_amount REAL,
  subsidy_ratio REAL,
  sample_size INT,
  province_prices JSONB,
  yearly_prices JSONB,
  updated_at TIMESTAMP
);
CREATE INDEX idx_subsidy_model ON subsidy_model_price(model);
CREATE INDEX idx_subsidy_category ON subsidy_model_price(category);
```

#### Phase 3：品牌档次系数自动化

从补贴数据按厂家聚合计算，替换手动 BRAND_COEFFICIENTS：

```python
# 伪代码
for manufacturer in all_manufacturers:
    models = data.filter(manufacturer=manufacturer)
    avg_price = mean(models.median_price)
    category_avg = mean(all_data.median_price for same categories)
    coefficient = avg_price / category_avg  # 自动计算
    # 范围限制
    coefficient = clamp(coefficient, 0.50, 1.20)
```

## 七、风险与对策

| 风险 | 对策 |
|------|------|
| JSON文件8MB影响冷启动 | Vercel Edge函数可能超限 → 用PostgreSQL表替代 |
| 补贴数据型号名与产品库不匹配 | 模糊匹配 + 品类回退 + 手动映射表 |
| V5置信度过高（样本量大） | 设上限0.95，加入AI视觉校验 |
| 国产品牌isChineseBrand未全部标注 | 回退到品牌系数判定（0.70分水岭）|
| 补贴数据时效性 | 标注dataFreshness，每年更新一次 |
